# Лабораторная работа 3.1: Поиск и изучение протоколов

## Задача
Изучить модель OSI и стек протоколов TCP/IP.

## Ход работы

В ходе выполнения лабораторной работы был проведен анализ стека протоколов TCP/IP в разрезе уровней модели OSI. Ниже представлено описание ключевых протоколов, их назначение и ссылки на соответствующие RFC.

---

## 1. Уровень доступа к сети (L1 и L2 модели OSI / Network Access Layer)

Этот уровень объединяет физический и канальный уровни. Он отвечает за передачу данных между двумя устройствами в одной сети.

### Протокол PPP (Point-to-Point Protocol)
*   **Описание:** Протокол канального уровня для установления прямой связи между двумя узлами. Поддерживает аутентификацию, шифрование и сжатие данных.
*   **RFC:** [1661](https://www.rfc-editor.org/rfc/rfc1661)

### Протокол PPPoE (Point-to-Point Protocol over Ethernet)
*   **Описание:** Инкапсуляция кадров PPP внутри Ethernet-кадров. Широко используется в xDSL-сервисах для предоставления дополнительных возможностей (аутентификация).
*   **RFC:** [2516](http://rfc.com.ru/rfc2516.htm)

### Протокол ARP (Address Resolution Protocol)
*   **Описание:** Используется для определения MAC-адреса устройства по его известному IP-адресу в локальной сети.
*   **RFC:** [826](https://rfc.com.ru/rfc826.htm)

### DSL (Digital Subscriber Line)
*   **Описание:** Технология, повышающая пропускную способность медных телефонных линий для одновременной передачи голоса и данных на высокой скорости.
*   **RFC:** [4319](https://www.rfc-editor.org/rfc/rfc4319.html), [3728](https://www.rfc-editor.org/rfc/rfc3728.html)

### Протокол MPLS (Multiprotocol Label Switching)
*   **Описание:** Механизм передачи данных с помощью меток, независимый от протоколов нижележащих уровней. Обеспечивает высокую производительность и масштабируемость в телекоммуникационных сетях.
*   **RFC:** [3031](http://www.protokols.ru/WP/rfc3031/)

### Протокол ATM (Asynchronous Transfer Mode)
*   **Описание:** Технология коммутации и мультиплексирования, использующая ячейки фиксированного размера (53 байта) для высокоскоростной передачи данных.
*   **RFC:** [1932](https://www.rfc-editor.org/rfc/rfc1932.html)

### Протокол Ethernet IEEE 802.2 (LLC)
*   **Описание:** Определяет управление логическим каналом (LLC), обеспечивая единый интерфейс для сетевого уровня поверх технологий IEEE 802.
*   **RFC:** [1700](https://www.rfc-editor.org/rfc/rfc1700) (включает информацию о стандартах)

### Протокол Ethernet IEEE 802.3
*   **Описание:** Семейство стандартов, определяющих проводные соединения, формат кадров и управление доступом к среде. Наиболее распространенная технология ЛВС.
*   **RFC:** (Стандарты IEEE, см. [IEEE 802.3](https://www.ieee.org))

### Протокол Token Bus IEEE 802.4
*   **Описание:** Технология локальной сети с широкополосной шиной и маркерным доступом. Использовалась в промышленных условиях (MAP) из-за устойчивости к помехам.
*   **RFC:** [1230](https://www.rfc-editor.org/rfc/rfc1230.html)

### Протокол Token Ring IEEE 802.5
*   **Описание:** Протокол ЛВС с топологией кольца и маркерным доступом, где данные передаются последовательно от одной станции к другой.
*   **RFC:** [1231](https://datatracker.ietf.org/doc/html/rfc1231)

### Технология FDDI (Fiber Distributed Data Interface)
*   **Описание:** Первая технология ЛВС на оптоволокне, обеспечивающая скорость 100 Мбит/с и суммарную длину сети до 100 км.
*   **RFC:** [1390](https://www.rfc-editor.org/rfc/rfc1390.html)

### Протокол ISDN (Integrated Services Digital Network)
*   **Описание:** Технология для передачи цифрового сигнала (включая голос и данные) по телефонным линиям со скоростью до 64 кбит/с.
*   **RFC:** [4715](https://datatracker.ietf.org/doc/html/rfc4715)

### Протокол Wi-Fi IEEE 802.11
*   **Описание:** Набор стандартов для беспроводной локальной сети (WLAN). Широко известен под брендом Wi-Fi.
*   **Ссылки:** [Wikipedia IEEE 802.11](https://ru.wikipedia.org/wiki/IEEE_802.11), [IEEE](https://www.ieee.org/searchresults/index.html?q=802.11#gsc.tab=0&gsc.q=802.11&gsc.page=1)

### Протокол Bluetooth IEEE 802.15
*   **Описание:** Технология беспроводной передачи данных на короткие расстояния для соединения различных устройств (телефоны, гарнитуры, периферия).

## 2. Интернет-уровень (L3 модели OSI / Internet Layer)

Отвечает за адресацию и маршрутизацию пакетов между различными сетями.

### Протокол IPv4 (Internet Protocol version 4)
*   **Описание:** Четвертая версия IP. Использует 32-битные адреса, что позволяет теоретически адресовать около 4.3 млрд устройств.
*   **RFC:** [791](https://rfc.com.ru/rfc791.htm)

### Протокол IPv6 (Internet Protocol version 6)
*   **Описание:** Новая версия IP с 128-битными адресами, разработанная для решения проблемы нехватки адресов IPv4.
*   **RFC:** [8200](http://www.protokols.ru/WP/rfc8200/)

### Протокол ICMP (Internet Control Message Protocol)
*   **Описание:** Протокол для передачи сообщений об ошибках и диагностической информации (например, недоступность хоста).
*   **RFC:** [792](https://rfc.com.ru/rfc792.htm)

### Протокол RIP (Routing Information Protocol)
*   **Описание:** Простой протокол маршрутизации (дистанционно-векторный), применяемый в небольших сетях. Маршрутизаторы обмениваются таблицами маршрутизации с соседями.
*   **RFC:** [2453](https://rfc.com.ru/rfc2453.htm)

### Протокол OSPF (Open Shortest Path First)
*   **Описание:** Протокол динамической маршрутизации (состояния каналов), использующий алгоритм Дейкстры для нахождения кратчайшего пути. Более эффективен, чем RIP, для крупных сетей.
*   **RFC:** [2328](https://rfc.com.ru/rfc2328.htm)

### Протокол NAT (Network Address Translation)
*   **Описание:** Механизм преобразования IP-адресов в пакетах (обычно частных в публичные), позволяющий нескольким устройствам выходить в интернет через один публичный адрес.
*   **RFC:** [3022](https://datatracker.ietf.org/doc/html/rfc3022)

## 3. Транспортный уровень (L4 модели OSI / Transport Layer)

Обеспечивает доставку данных между приложениями на хостах.

### Протокол TCP (Transmission Control Protocol)
*   **Описание:** Основной протокол с установлением соединения, гарантирующий надежную доставку данных в нужном порядке и без потерь.
*   **RFC:** [793](https://rfc.com.ru/rfc793.htm)

### Протокол UDP (User Datagram Protocol)
*   **Описание:** Протокол без установления соединения, не гарантирующий доставку. Используется там, где важна скорость, а не надежность (видеозвонки, DNS).
*   **RFC:** [768](https://www.lissyara.su/doc/rfc/rfc768/)

### Протокол RTCP (Real-Time Transport Control Protocol)
*   **Описание:** Работает совместно с RTP для мониторинга качества передачи (информация о задержках, потерях пакетов, джиттере).
*   **RFC:** [3550](https://www.rfc-editor.org/rfc/rfc3550)

### Протокол SCTP (Stream Control Transmission Protocol)
*   **Описание:** Современный транспортный протокол, сочетающий свойства TCP (надежность) и UDP (скорость) с поддержкой мультипоточности и multi-homing.
*   **RFC:** [4960](https://www.rfc-editor.org/rfc/rfc4960.html), [3286](https://rfc.com.ru/rfc3286.htm)

### Протокол FCP (Fibre Channel Protocol)
*   **Описание:** Транспортный протокол для инкапсуляции команд SCSI в сетях Fibre Channel. Основа для построения сетей хранения данных (SAN).

### Протокол DCCP (Datagram Congestion Control Protocol)
*   **Описание:** Протокол транспортного уровня, предоставляющий механизмы для контроля перегрузок в сети без гарантии доставки и порядка сообщений.
*   **RFC:** [4340](https://rfc.com.ru/rfc4340.htm)

## 4. Уровень приложений (L5-7 модели OSI / Application Layer)

Взаимодействует напрямую с приложениями пользователя.

### Протокол HTTP (HyperText Transfer Protocol)
*   **Описание:** Основа веба. Протокол "запрос-ответ" для передачи гипертекста и других данных.
*   **RFC:** [1945](https://www.rfc-editor.org/rfc/rfc1945) (HTTP/1.0), [2616](https://www.rfc-editor.org/rfc/rfc2616) (HTTP/1.1), [7240](https://www.rfc-editor.org/rfc/rfc7240) (Prefer Header)

### Протокол FTP (File Transfer Protocol)
*   **Описание:** Один из старейших протоколов для передачи файлов между клиентом и сервером.
*   **RFC:** [959](https://www.rfc-editor.org/rfc/rfc959)

### Протокол POP3 (Post Office Protocol Version 3)
*   **Описание:** Протокол для получения электронной почты клиентом с удаленного сервера (обычно с последующим удалением).
*   **RFC:** [1939](https://www.ietf.org/rfc/rfc1939.txt), [5034](https://datatracker.ietf.org/doc/html/rfc5034)

### Протокол SMTP (Simple Mail Transfer Protocol)
*   **Описание:** Основной протокол для отправки электронной почты между серверами и от клиента на сервер.
*   **RFC:** [5321](https://www.rfc-editor.org/rfc/rfc5321)

### Протокол RDP (Remote Desktop Protocol)
*   **Описание:** Проприетарный протокол Microsoft для удаленного доступа к рабочему столу компьютера.
*   **RFC:** [908](https://datatracker.ietf.org/doc/html/rfc908), [1151](https://datatracker.ietf.org/doc/html/rfc1151)

### Протокол SMB (Server Message Block)
*   **Описание:** Протокол для удаленного доступа к файлам, принтерам и другим сетевым ресурсам (файловые расшаренные папки в Windows).

### Протокол RTP (Real-time Transport Protocol)
*   **Описание:** Используется для доставки аудио и видео в реальном времени (VoIP, видеоконференции). Переносит информацию о типе кодировки и временных метках.
*   **RFC:** [4571](https://www.rfc-editor.org/rfc/rfc4571) (для протоколов поверх RTP)

### Протокол TELNET (teletype network)
*   **Описание:** Протокол для текстового терминального доступа к удаленному хосту. Передает данные в незашифрованном виде.
*   **RFC:** [5198](https://www.rfc-editor.org/rfc/rfc5198)

### Протокол SIP (Session Initiation Protocol)
*   **Описание:** Сигнальный протокол для установления, изменения и завершения мультимедийных сеансов связи (например, IP-телефония).
*   **RFC:** [3261](https://www.rfc-editor.org/rfc/rfc3261)

### Протокол XDR (External Data Representation)
*   **Описание:** Стандарт для представления данных, независимый от архитектуры компьютера. Используется для обмена данными в гетерогенных средах (например, в RPC).
*   **RFC:** [4506](https://www.rfc-editor.org/rfc/rfc4506)

### Протокол LPP (Lightweight Presentation Protocol)
*   **Описание:** Облегченный протокол уровня представления для предоставления сервисов OSI поверх TCP/IP.
*   **RFC:** [1085](https://www.rfc-editor.org/rfc/rfc1085.html)

### Протокол NDR (Network Data Representation)
*   **Описание:** Синтаксис передачи данных, используемый в Microsoft RPC для определения порядка и выравнивания данных при передаче по сети.
*   **RFC:** [3464](https://www.rfc-editor.org/rfc/rfc3464) (в контексте DSN)

### Протокол PAP (Password Authentication Protocol)
*   **Описание:** Простой протокол аутентификации, передающий имя пользователя и пароль в открытом виде (используется в PPP).
*   **RFC:** [2865](https://www.rfc-editor.org/rfc/rfc2865) (RADIUS, частью которого является)

### Протокол L2TP (Layer 2 Tunnelling Protocol)
*   **Описание:** Туннельный протокол для поддержки виртуальных частных сетей (VPN). Может работать не только поверх IP, но и ATM, Frame Relay.
*   **RFC:** [2661](https://www.rfc-editor.org/rfc/rfc2661.html)

### Протокол SMPP (Short Message Peer-to-Peer)
*   **Описание:** Открытый протокол для обмена SMS-сообщениями между различными телекоммуникационными платформами и центрами служб коротких сообщений (SMSC).
*   **RFC:** [6118](https://www.rfc-editor.org/rfc/rfc6118.html)

### Протокол SDP (Session Description Protocol)
*   **Описание:** Протокол для описания параметров мультимедийных сессий (кодеки, IP-адреса, порты), используемый совместно с SIP или RTSP.
*   **RFC:** [4566](https://www.rfc-editor.org/rfc/rfc4566)

### Протокол SCP (secure copy)
*   **Описание:** Утилита и протокол для безопасного копирования файлов между компьютерами, использующая SSH для шифрования.
*   **RFC:** [2995](https://www.rfc-editor.org/rfc/rfc2995) (в контексте RSIP)

### Протокол PPTP (Point-to-Point Tunneling Protocol)
*   **Описание:** Устаревший туннельный протокол для создания VPN, инкапсулирующий кадры PPP в IP-пакеты.
*   **RFC:** [2637](https://www.rfc-editor.org/rfc/rfc2637)
